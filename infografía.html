GastroCloud

Startup Case Study

Una startup que busca crear una app revolucionaria para descubrir restaurantes. Necesitan una arquitectura robusta desde el día uno.

Requisitos de Negocio

  • Rápida y escalable (millones de usuarios).
  • Subida de fotos y reseñas de platos.
  • Recomendaciones con IA y análisis de sentimientos.
  • Seguridad de datos y control de costos.

El Desafío: 6 Módulos, 6 Decisiones

Trabajo en Grupo
Módulo 1: Transformación

OpEx vs CapEx

¿Por qué la nube (OpEx) es vital para esta startup en lugar de comprar servidores (CapEx)?

Módulo 4: Modernización

Plataforma de Cómputo

Para el backend: ¿VMs, GKE o Cloud Run/Serverless? Justifique su elección.

Módulo 2: Datos

Almacenamiento de Fotos

¿Qué tipo de datos son las fotos? ¿Usarían Cloud SQL o Cloud Storage?

Módulo 3: IA

Análisis de Sentimiento

Para las reseñas: ¿API pre-entrenada, AutoML o Vertex AI?

Módulo 5: Seguridad

Control de Acceso

¿Cómo asegurar que solo el usuario edite su perfil? (Principio de Menor Privilegio).

Módulo 6: Operaciones

Control de Costos (FinOps)

¿Cómo evitar que los costos de IA se disparen? ¿Etiquetas, Presupuestos o Cuotas?

Guía para el Debate y Cierre

Puntos de Discusión

  • Contrastar decisiones (ej. Grupo A eligió VMs vs Grupo B eligió Cloud Run).
  • Discutir Pros y Contras de Serverless para una Startup.
  • Validar elección de Cloud Storage para objetos no estructurados (fotos).
  • Reforzar la conexión: Modernización afecta Costos; Datos alimentan la IA.

Mensaje de Cierre

"Hoy demostraron que no solo saben los conceptos, sino que pueden usarlos para construir una solución. Han dejado de ver los árboles para diseñar el bosque."
